Server Side programming

Web programming Server Side is the practice of writing applications that run on a web server and can be used by many different people.

Web programming allows you to turn a simple, static HTML page into a dynamic masterpiece.

  • It allows others to interact with your web site and use the application on any computer with Internet access.
  • It is often easier than programming applications that will run directly on the computer.
  • It allows you to make or edit anything dynamic on your website, such as a forum, a guestbook, or even a form submission.

    • This course will help you understand, what Server Side Web Programming is

La Programmation côté serveur

La programmation Web côté serveur consiste à écrire des applications qui s'exécutent sur un serveur Web et peuvent être utilisées par de nombreuses personnes différentes.

La programmation Web vous permet de transformer une simple page HTML statique en un chef-d'œuvre dynamique.

  • Il permet à d'autres personnes d'interagir avec votre site Web et d'utiliser l'application sur n'importe quel ordinateur ayant accès à Internet.
  • C'est souvent plus simple que de programmer des applications qui s'exécuteront directement sur l'ordinateur.
  • Il vous permet de créer ou de modifier quoi que ce soit de dynamique sur votre site Web, comme un forum, un livre d'or ou même une soumission de formulaire.

    • Ce cours vous aidera à comprendre ce qu'est la programmation Web côté serveur.

Web Server

The term web server can refer to either the hardware (the computer) or the software (the computer application) that helps to deliver web content that can be accessed through the Internet. The most common use of web servers is to host websites, but there are other uses such as gaming, data storage or running enterprise applications.

  • IIS - Internet Information Services
    • Microsoft Windows
  • Apache Web Server
    • Open Source
    • Cross-platform: UNIX, Linux, OS X, Windows, ...
  • Nginx (pronounced "engine x") - Has become very popular latly
  • ...

Serveur Web

Le terme serveur Web peut désigner soit le matériel (l'ordinateur), soit le logiciel (le application informatique) qui permet de fournir du contenu Web accessible via le L'Internet. L'utilisation la plus courante des serveurs Web consiste à héberger des sites Web, mais il existe d'autres utilisations telles que les jeux, le stockage de données ou l'exécution d'applications d'entreprise.

PHP Installation

De quoi ai-je besoin ?

Pour commencer à utiliser PHP, vous pouvez :

  • Trouvez un hébergeur Web prenant en charge PHP et MySQL
  • Installez un serveur Web sur votre propre PC, puis installez PHP et MySQL

Use a Web Host With PHP Support

If your server has activated support for PHP you do not need to do anything.

Just create some .php files, place them in your web directory, and the server will automatically parse them for you.

You do not need to compile anything or install any extra tools.

Because PHP is free, most web hosts offer PHP support.

Utiliser un hébergeur Web avec support PHP

Si votre serveur a activé la prise en charge de PHP, vous ne besoin de faire quelque chose.

Créez simplement des fichiers .php, placez-les dans votre répertoire Web et le serveur les analysera automatiquement pour vous.

Vous n'avez pas besoin de compiler quoi que ce soit ou d'installer des outils supplémentaires.

Parce que PHP est gratuit, la plupart des hébergeurs offrir un support PHP.

Introduction to XAMPP

XAMPP is the most popular software package which is used to set up a PHP development environment for web services by providing all the required software components. During the process of software deployment, most of the web servers use almost similar components, so use of XAMPP provides easy transition from local server to live server. XAMPP is a AMP stack which stands for Cross platform, Apache, MySQL, PHP, perl with some additional administrative software tools such as PHPMyAdmin (for database access), FileZilla FTP server, Mercury mail server and JSP Tomcat server. Other commonly known software packages like XAMPP are WAMP, LAMP, and others.
The XAMPP server is used to test PHP pages. It works as local server. It contains a MySQL database to manage or save data on a local server.

Présentation de XAMPP

XAMPP est le progiciel le plus populaire utilisé pour configurer un environnement de développement PHP pour les services Web en fournissant tous les composants logiciels requis. Au cours du processus de déploiement de logiciels, la plupart des serveurs Web utilisent des composants presque similaires. L'utilisation de XAMPP permet donc une transition facile du serveur local au serveur en direct. XAMPP est une pile AMP qui signifie Cross platform, Apache, MySQL, PHP, perl avec quelques outils logiciels d'administration supplémentaires tels que PHPMyAdmin (pour l'accès à la base de données), le serveur FTP FileZilla, le serveur de messagerie Mercury et le serveur JSP Tomcat. D'autres progiciels connus comme XAMPP sont WAMP, LAMP et autres.
Le serveur XAMPP est utilisé pour tester les pages PHP. Il fonctionne comme serveur local. Il contient une base de données MySQL pour gérer ou enregistrer des données sur un serveur local.

Software components of XAMPP:

  • Apache plays the role of processing the HTTP request. It is the actual default web server application. It is the most popular web servers maintained by Apache Software Foundation.
  • MySQL The role of database management system in XAMPP is played by MySQL. It helps to store and manage collected data very efficiently. It is an open-source and most popular.
  • PHP is the server-side scripting language which stand for Hypertext Preprocessor. It is embedded with HTML code which interacts with the webserver. It is an open-source and work well with MySQL and has become a common choice for web developers.
  • Perl is the high-level programming language designed for text editing which serves purpose like web development and network programming.

Composants logiciels de XAMPP :

  • Apache joue le rôle de traitement de la requête HTTP. Il s'agit de l'application de serveur Web par défaut. Il s'agit des serveurs Web les plus populaires gérés par Apache Software Foundation.
  • MySQL Le rôle de système de gestion de base de données dans XAMPP est joué par MySQL. Il permet de stocker et de gérer très efficacement les données collectées. C'est une source ouverte et la plus populaire.
  • PHP est le langage de script côté serveur qui signifie Hypertext Preprocessor. Il est intégré au code HTML qui interagit avec le serveur Web. Il s'agit d'un open source qui fonctionne bien avec MySQL et est devenu un choix courant pour les développeurs Web.
  • Perl est le langage de programmation de haut niveau conçu pour l'édition de texte qui sert à des fins telles que le développement Web et la programmation réseau.

Steps to install XAMPP on Windows:

  • In the web browser, visit Apache Friends and download XAMPP installer.

Étapes pour installer XAMPP sous Windows :

  • Dans le navigateur Web, visitez Apache Friends et téléchargez XAMPP installateur.

  • During the installation process, select the required components like MySQL, FileZilla ftp server, PHP, phpMyAdmin or leave the default options and click the Next button.
  • Pendant le processus d'installation, sélectionnez les composants requis tels que MySQL, le serveur ftp FileZilla, PHP, phpMyAdmin ou laissez les options par défaut et cliquez sur le bouton Suivant.

  • Uncheck the Learn more about bitnami option and click Next button.
  • Choose the root directory path to set up the htdocs folder for our applications. For example ‘C:\xampp’.
  • Click the Allow access button to allow the XAMPP modules from the Windows firewall.
  • After the installation process, click the Finish button of the XAMPP Setup wizard.
  • Now the XAMPP icon is clearly visible on the right side of start menu. Show or Hide can be set by using the control panel by clicking on the icon.
  • To start Apache and MySql, just click on the Start button on the control panel.
  • Décochez l'option En savoir plus sur bitnami et cliquez sur le bouton Suivant.
  • Choisissez le chemin du répertoire racine pour configurer le dossier htdocs pour nos applications. Par exemple ‘C:\xampp’.
  • Cliquez sur le bouton Autoriser l'accès pour autoriser les modules XAMPP à partir du pare-feu Windows.
  • Après le processus d'installation, cliquez sur le bouton Terminer de l'assistant de configuration XAMPP.
  • Maintenant, l'icône XAMPP est clairement visible sur le côté droit du menu Démarrer. Afficher ou Masquer peut être défini à l'aide du panneau de configuration en cliquant sur l'icône.
  • Pour démarrer Apache et MySql, cliquez simplement sur le bouton Démarrer du panneau de configuration.

Note: Suppose Apache is not starting, it means some other service is running at port 80. In this case, stop the other service temporarily and restart it.
Making server request: Open your web browser and check whether the XAMPP service has properly installed or not. Type in the URL: http://localhost. If you are able to see the default page for XAMPP, you have successfully installed your XAMPP Server.
To Check if PHP is Working: All the website related files are organized in a folder called htdocs and then run index.php file by using http://localhost/index.php or http://localhost.
Note: For every new website or application, its always better to create a different folder inside htdocs, to keep it organized and avoid confusion.

Remarque : Supposons qu'Apache ne démarre pas, cela signifie qu'un autre service est en cours d'exécution sur le port 80. Dans ce cas, arrêtez temporairement l'autre service et redémarrez-le.
Faire une requête au serveur : Ouvrez votre navigateur Web et vérifiez si le service XAMPP est correctement installé ou non. Saisissez l'URL : http://localhost. Si vous pouvez voir la page par défaut pour XAMPP, vous avez installé avec succès votre serveur XAMPP.
Pour vérifier si PHP fonctionne : Tous les fichiers liés au site Web sont organisés dans un dossier appelé htdocs, puis exécutez le fichier index.php en utilisant http://localhost /index.php ou http://localhost.
Remarque : Pour chaque nouveau site Web ou application, il est toujours préférable de créer un dossier différent dans htdocs, pour le garder organisé et éviter toute confusion.
