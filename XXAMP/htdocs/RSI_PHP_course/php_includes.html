PHP Include Files

The include (or require) statement takes all the text/code/markup that exists in the specified file and copies it into the file that uses the include statement.

Including files is very useful when you want to include the same PHP, HTML, or text on multiple pages of a website.

PHP Inclure les fichiers

L'instruction include (ou require) prend tout le texte/ code/balisage qui existe dans le fichier spécifié et le copie dans le fichier qui utilise l'instruction include.

L'inclusion de fichiers est très utile lorsque vous souhaitez inclure le même PHP, HTML ou texte sur plusieurs pages d'un site Web.

PHP include and require Statements

It is possible to insert the content of one PHP file into another PHP file (before the server executes it), with the include or require statement.

The include and require statements are identical, except upon failure:

  • require will produce a fatal error (E_COMPILE_ERROR) and stop the script
  • include will only produce a warning (E_WARNING) and the script will continue

So, if you want the execution to go on and show users the output, even if the include file is missing, use the include statement. Otherwise, in case of FrameWork, CMS, or a complex PHP application coding, always use the require statement to include a key file to the flow of execution. This will help avoid compromising your application's security and integrity, just in-case one key file is accidentally missing.

Including files saves a lot of work. This means that you can create a standard header, footer, or menu file for all your web pages. Then, when the header needs to be updated, you can only update the header include file.

Syntax

PHP inclut et exige des déclarations

Il est possible d'insérer le contenu d'un fichier PHP dans un autre fichier PHP (avant serveur l'exécute), avec l'instruction include ou require.

Les instructions include et require sont identiques, sauf en cas d'échec :

  • require produira une erreur fatale (E_COMPILE_ERROR) et arrêtera le script
  • include ne produira qu'un avertissement (E_WARNING) et le script continuera

Donc, si vous voulez que l'exécution se poursuive et montre aux utilisateurs la sortie, même si le fichier include est manquant, utilisez l'instruction include. Sinon, en cas de FrameWork, de CMS ou d'un codage d'application PHP complexe, utilisez toujours l'instruction require pour inclure un fichier clé dans flux d'exécution. Cela vous évitera de compromettre la sécurité de votre application et l'intégrité, juste au cas où un fichier clé serait accidentellement manquant.

L'inclusion de fichiers permet d'économiser beaucoup de travail. Cela signifie que vous pouvez créer un en-tête, un pied de page ou un fichier de menu standard pour toutes vos pages Web. Ensuite, lorsque l'en-tête doit être mis à jour, vous pouvez uniquement mettre à jour le fichier d'en-tête inclus.

Syntaxe

include 'filename';
or

require 'filename';

PHP include vs. require

The require statement is also used to include a file into the PHP code.

However, there is one big difference between include and require; when a file is included with the include statement and PHP cannot find it, the script will continue to execute:

PHP inclure vs exiger

L'instruction require est également utilisée pour inclure un fichier dans le code PHP.

Cependant, il existe une grande différence entre inclure et exiger ; lorsqu'un le fichier est inclus avec l'instruction include et PHP ne peut pas le trouver, le script continuera à s'exécuter :

Example

<html>
<body>

<h1>Welcome to my home page!</h1>
<?php include 'noFileExists.php';
echo "I have a $color $car.";

?>

</body>
</html>

If we do the same example using the require statement, the echo statement will not be executed because the script execution dies after the require statement returned a fatal error:

Si nous faisons le même exemple en utilisant l'instruction require, le l'instruction echo ne sera pas exécutée car l'exécution du script meurt après la L'instruction require a renvoyé une erreur fatale :

Example

<html>
<body>

<h1>Welcome to my home page!</h1>
<?php require 'noFileExists.php';
echo "I have a $color $car.";
?>

</body>
</html>

Use require when the file is required by the application.

Use include when the file is not required and application should continue when file is not found.



Utilisez require lorsque le fichier est requis par l'application.

Utilisez include lorsque le fichier n'est pas requis et l'application doit continuer lorsque le fichier n'est pas trouvé.
